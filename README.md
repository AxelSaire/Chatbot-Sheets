
# CHATBOT DM

MANUAL DE FUNCIONAMIENTO DE CHATBOT PARA DM MECHATRONICS



## About
CHATBOT PARA REGISTRO AUTOMATICO DE DATOS A NIVEL ADMINISTRATIVO. IMPLEMENTADO USANDO LA LIBRERIA DE BAILEYS EN SU VERSION 6.6.0
## Installation

DESCARGAR E INSTALAR NODEJS MOTOR PRINCIPAL DE EJECUCION DEL PROGRAMA

```bash
  https://nodejs.org/dist/v24.15.0/node-v24.15.0-x64.msi
```
INSTALAR LAS DEPENDENCIAS NECESARIAS PARA EL PROGRAMA
```bash
  npm i @whiskeysockets/baileys@6.6.0
  npm i pino
  npm i googleapis
  npm i fs
  npm install -g pm2
```

## Credentials
PARA GENERAR CREDENCIALES VISITAR EL SIGUIENTE ENLACE

PARA LA EJECUCION DEL PROGRAMA EJECUTAR EN TERMINAL LOS SIGUIENTES COMANDOS

```bash
pm2 start index.js --name "whatsapp-bot"
pm2 save

```


## Herramientas
DENTRO DEL CONJUNDO DE FUNCIONES QUE CONTEMPLA EL BOT ESTAN:
### REGISTRO DE SERVICIOS
PARA EL USO CORRECTO DE LA FUNCION SE DEBE COLOCAR NECESARIAMENTE LOS DATOS DE LA SIGUIENTE MANERA:
```
USUARIO, EQUIPO, PRECIO, DESCRIPCION
¡! IMPORTANTE PARA REGISTRAR UN PRECIO EN SOLES ANTEPONER LA 's' DELANTE DEL PRECIO
```
SIENDO LOS TRES PRIMEROS DATOS OBLIGATORIOS PARA QUE LOS DATOS SE REGISTREN SATISFACTORIAMENTE, EL CUARTO DATO ES OPCIONAL Y SIGUE LA SIGUIENTE LOGICA:
```
SI PRECIO > 350 USD ENTONCES 'CONFIGURACION'
CASO CONTRARIO SI PRECIO ES MENOR A 350  ENTONCES 'EMISIONES'
```
EN CASO DE PAGOS CON MONEDA NACIONAL (SOLES PERUANOS) SIGUE LA SIGUIENTE LOGICA:
```
SIN DESCRIPCION ENTONCES 'EMISIONES'
```
EN CASO DE QUE SE INTRODUZCA EL CUARTO PARAMETRO (DESCRIPCION) SE RESPETAN LOS DATOS INTRODUCIDOS, EJEMPLOS DE USO
```
INPUT
JUAN PEREZ, MAQUINA 1, 500
RODRIGO GONZALES, MAQUINA 2, 100, WASH
HUGO RAMIREZ, MAQUINA 3, s400
JOSE LOPEZ, MAQUINA 4, s200, INTEGRAL

OUTPUT
|CLIENTE| EQUIPO| PRECIO| DESCRIPCION DE SERVICIO|
|JUAN PEREZ| MAQUINA 1| $500| CONFIGURACION|
|RODRIGO GONZALES| MAQUINA 2| $100| WASH|
|HUGO RAMIREZ| MAQUINA 3| S/. 400| EMISIONES|
|JOSE LOPEZ| MAQUINA 4| S/. 200| INTEGRAL|

```
**IMPORTANTE:** EL ESTADO DE SERVICIO EN CASO DE REGISTRO, SE REGISTRAN COMO *PENDIENTE* A MODIFICAR A NIVEL CONTABLE EN EL DOCUMENTO

EJEMPLO:
EN CASO DE QUE 'JUAN PEREZ' TENGA UN SERVICIO 'PENDIENTE' ENTONCES AL MOMENTO DE REGISTRAR, LANZA ADVERTENCIA DE REGISTRO PENDIENTE DE PAGO Y LO REGISTRAN

```
EN SHEETS
|JUAN PEREZ| MAQUINA 1| $500| CONFIGURACION|PENDIENTE|

INPUT
JUAN PEREZ, MAQUINA 5, $200

OUTPUT
'!ADVERTENCIA! JUAN PEREZ TIENE UNA OPERACION PENDIENTE DE PAGO' Y REGISTRA

|JUAN PEREZ| MAQUINA 1| $500| CONFIGURACION|PENDIENTE|
|JUAN PEREZ| MAQUINA 1| $200| CONFIGURACION|PENDIENTE|
```
### OTRAS FUNCIONES
PARA EL USO DE FUNCIONES USAMOS '/' COMO DELIMITADOR DE FUNCION
EJEMPLO RESUMEN
```
\resumen hoy
DEVUELVE  Operaciones: Días activos, Total USD, Total Soles, Promedio diario
\resumen mes
\resumen year
```
EJEMPLO ELIMINAR ULTIMO REGISTRO
```
EN SHEETS 
|RUBEN HENRIQUE|MAQUINA 6|$300|EMISIONES|PENDIENTE| <---ULTIMO REGISTRO

INPUT
/eliminarultimo

OUTPUT
Eliminado correctamente
    👤 RUBEN HENRIQUE

| BLANK| BLANK| BLANK| BLANK| BLANK|
```
